# Siguientes skills que podríamos añadir

Estas son las skills que más valor podrían aportar si el repo crece.

## Prioridad alta

1. **backend**
   - Para aterrizar APIs, autenticación, integración con servicios y diseño de contratos.

2. **arquitectura**
   - Para documentar decisiones de estructura, capas, escalabilidad y tradeoffs.

3. **testing**
   - Para definir estrategia de unit, integration y e2e en cada stack.

## Prioridad media

4. **bases-de-datos**
   - Para modelado, persistencia local/remota y decisiones de almacenamiento.

5. **automatización**
   - Para bootstraps, scripts, CI/CD, releases y tareas repetibles.

6. **editor-ai-first**
   - Para adaptar el repo a flujos con editores IA-first como opencode, Cursor o similares.

## Prioridad contextual

7. **wsl-en-windows**
   - Útil si se quiere formalizar la recomendación de correr opencode en Windows bajo WSL2.

8. **documentación-operativa**
   - Para mantener README, guías de uso y convenciones siempre alineadas.

## Nota

La prioridad real debería ser: primero las skills que ayudan a pensar mejor, luego las que ayudan a ejecutar más rápido.
